Monetary requirements established by the buyer related to a procurement.<br/><br/>Additional Information:<br/><br/>A value given for a group of lots may be different to the sum of the lots submitted individually (e.g. more advantageous).<br/>Separate classes are added in the ontology for different value types (e.g. Procurement Value, Financial Official Value, other) for tracking back the provenance and enabling the individual and total values to be cross-checked.<br/><br/>WG Approval 28/05/2019<br/>

LotGroup
Class   		Name: has Estimated Value
  		Name:  
 
Details:
A forecast of the value of the procurement before competition.<br/><br/>Additional Information<br/>Different cases of estimated values may refer to a lot, the global value of the procedure, or of a combinatorial value of a group of lots.<br/>The forecast is calculated by the buyer and covers all revenues  whether coming from the buyer or third parties.<br/>See for example recital (19), Article 5 of Directive 2014/24/EU and other articles from the rest of Directives about procurement.<br/><br/>In the case of lots or groups of lots using framework agreements and dynamic purchasing systems this refers to the maximum estimated value.<br/><b><br/></b><b>WG Approval 05/12/2019</b><br/>

Contract Award Notice

  		Name: hasTotalFrameworkValue
 
Details:
The maximum value which can be spent through all the framework agreements announced in this notice, including options and renewals of contracts.<br/><br/>Additional information:<br/><br/>The value provided is a threshold value that implicity means that it cannot be exceeded however it may not be reached during the execution of a contract.<br/><br/>The Framework Agreements in a CAN are to be traced back and added to provide this value. The path to get the value of an individual Framework Agreement is:<br/><br/>CAN isA Notice,<br/>CAN notifiesAwardResult AwardDecision,<br/>AwardDecision refersTo Lot,<br/>Lot uses Technique, <br/>FrameworkAgreement isA Technique,<br/>FrameworkAgreement hasAwardedValue ProcurementValue .<br/><br/>WG Approval 03/12/2019<br/><br/><br/>
