Notes: The property Type can be used to identify the namespace and class of the Agent.<br/><br/>This may be useful for XML implementations of the Model, where software applications need to know the type of the Agent in order to identify the particular Agent sub-class properties. But could also create a consistency problem consisting in specifying an IRI for one type of Agent whilst the actual individual of Agent is a different one.<br/><br/>One way of resolving this inconsistency would be cross-checking by externalised means the requested IRI ID against the actual type of class (via shapes, Schematron, etc.).<br/><br/>For more details on IRIs, check the Request For Comments website: https://tools.ietf.org/html/rfc3987<br/><br/>Examples of IRIs:<br/><br/><font color="#993300">https://github.com/semiceu/cagv/2.0.0/xml/CV-Agent#Organisation</font><br/><font color="#993300"><br/></font><font color="#993300">https://github.com/semiceu/cagv/2.0.0/xml/CV-Agent#NaturalPerson</font><br/><font color="#993300"><br/></font><font color="#993300">https://github.com/semiceu/cagv/2.0.0/xml/CV-Agent#System</font><br/><br/>An example of inconsistency:<br/><br/>cagv:Agent.ID = 986454b6-22af-4894-9a3d-f201d236b88c<br/>cagv:Agent.Type = <font color="#993300">https://github.com/semiceu/cagv/2.0.0/xml/CV-Agent#ORganisation</font><br/>cagv:NaturalPerson.ID = 986454b6-22af-4894-9a3d-f201d236b88c<br/><br/><br/><br/>
